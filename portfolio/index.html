AZ Magazine Storefront Overview

AZ brings a slice of that fascinating world to your iPad, letting readers tap into stories from Arizona and around the world by quite literally tapping - and scrolling, swiping and "scrubbing" - using interactive elements to bring stories to life. This format is undeniably cool, but AZ will deliver more than pretty things to look at. Our stories aim to inform, to uplift, to captivate... and to remind you that the world is just a little bigger than you thought.

My Responsibilities

  • Convert flat design mockup images into code
  • Write all storefront application functionality.

Development

Requirements were documented in Basecamp. The complete tablet app would be built using the Adobe Digital Publishing Suite, so the storefront needed to be created using web technologies. I was given a single flat image design mockup:

Functionally, the storefont needed to use the Adobe DPS API to access data about individual folios and user account. I used Backbone.js to structure and render the storefont and some jQuery for some visual interactivity.

The functional requirements included a basic slideshow for a small set of images. Rather than include the typical Galleria based gallery, I created a lightweight jQuery based slideshow.

Explore AZ Mobile Overview

"For nearly 500 more insider tips from Republic experts on the top places to eat, play and stay around the state, browse our Top 10 Explore Arizona mobile experience."

My Responsibilities

  • Convert flat design mockup images into code
  • Write all web application functionality.
  • Have version 1 ready quickly for beta test

Development

Requirements were documented in Basecamp. I was given flat image design mockups. An example of these mockups:

The initial version was needed quickly for use by an external test group, so I opted to use jQuery Mobile for the application framework. Because I needed to covert image based mockups into an application quickly, I opted to use LESS to speed up the amount of CSS I had to write.

For the sake of user experience, I included functionality that leveraged the HTML5 Geolocation API to default a user's destination to their current location. I also included functionality that displayed relevant user reviews via the Google Places API and the location of the venue on a Google Map.
